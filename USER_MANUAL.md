# User Manual: Image Forensics SIEM Dashboard

## Introduction
The **Image Forensics SIEM (Security Information and Event Management) Dashboard** is an advanced, automated toolkit for analyzing image authenticity and detecting tampering, manipulation, or synthetic generation. It comprises three distinct engines—the Physics Engine, AI Engine, and OSINT Engine—that run synchronously to pinpoint anomalies and generate a unified threat report.

## System Requirements & Installation
Ensure you have the necessary dependencies installed:
```bash
pip install -r requirements.txt
```
To launch the application globally or within your local environment:
```bash
streamlit run app.py
```

## Getting Started

### Launching the Dashboard
Upon running the command above, a modernized, futuristic dark-themed web interface will open in your browser. The dashboard is divided into two primary areas:
1. **Case Management (Left Sidebar):** For evidence intake and analysis execution.
2. **Main Dashboard (Center Area):** Where all results, metrics, and visual maps are projected.

### How to Analyze an Image
1. Expand the **Case Management** tab on the left sidebar.
2. Click **Browse Files** to upload your Evidence Image. Supported formats include `.jpg`, `.jpeg`, `.png`, and `.webp`.
3. Once the image is uploaded, a thumbnail called **Evidence Subject** will appear.
4. Click the **🚀 Run Forensic Analysis** button to initialize the pipeline.
5. The system will synchronously run the Physics, AI, and OSINT engines. Wait for the success message: "Analysis Complete!".

## Interpreting the Dashboard Results

Once an image finishes processing, the Main Dashboard populates with detailed insights.

### 1. Executive Summary
A quick-glance metric board summarizes the analysis:
*   **Total Findings:** The aggregate amount of forensics alerts captured across all modules.
*   **Critical Anomalies (🚨):** High-severity issues that strongly suggest tampering or synthetic origin.
*   **Warnings (⚠️):** Medium-severity events indicating suspicious characteristics requiring human review.
*   **AI Confidence:** The generative AI confidence score, measuring the likeliness of the image being synthetic.

### 2. Visual Evidence Analysis
Displays your original, uploaded image. You can hover your cursor over the image to magnify specific regions (Zoom-in functionality).

### 3. Forensic Alert Timeline
An interactive feed displaying color-coded alerts indicating the type and severity of findings:
*   🔴 **High (Red):** Critical tampered regions, GenAI matching, Deepfake detections.
*   🟡 **Medium (Orange):** Missing/manipulated metadata, irregular noise distributions.
*   🔵 **Low/Info (Blue):** Standard logging, unmodified EXIF info, weather metadata.

**Interactive Visual Maps:**
For many alerts, an expander label (e.g., `🔍 Inspect PRNU Visual Map overlay`) lets you juxtapose the original image against pixel-level analytical maps generated by the physics engine.

## Core Engine Capabilities

### 1. Physics Engine
The physics engine breaks down image structure to expose pixel-level tampering:
*   **Error Level Analysis (ELA):** Identifies inconsistent compression qualities across the image. Spliced objects often look drastically different in an ELA map.
*   **PRNU (Photo Response Non-Uniformity):** Extracts Haar wavelet noise mappings. It detects anomalous sensor noise clusters to determine if multiple camera sensors acquired different parts of the image.
*   **Shadow / Luminance Grading:** Evaluates unnaturally smooth gradients that are often the hallmark of airbrushing or GAN (Generative Adversarial Network) outputs.
*   **JPEG Ghosting:** Looks for double quantization by re-saving the image and finding inconsistent differential compressions characteristic of cropped/pasted areas.

### 2. AI Engine
Powered by locally executed or remote classification models:
*   **GenAI Detection:** Uses SDXL-detector models to identify imagery artificially composed by Midjourney, Stable Diffusion, or similar generation models.
*   **Deepfake / Face Swapping:** Segregates faces and tests them against Deepfake-recognition weights, returning a direct confidence interval on the authenticity of the subject.

### 3. OSINT Engine (Open-Source Intelligence)
Extracts embedded traces and matches them with external data grids:
*   **Metadata Extraction:** Reads foundational EXIF profiles (Device Make, Device Model, Exposure time) and highlights traces of Software Editing (e.g., Photoshop, GIMP).
*   **GPS & Geolocation:** If GPS traits exist, the engine reverse-maps the geographic coordinates and generates an interactive map (`📍 View OSINT GPS Coordinates Map`).
*   **Historical Weather Verification:** Cross-references the coordinate and DateTime attributes with Open-Meteo's database. It checks to see if the weather conditions depicted in your image match the factual meteorological records of that specific hour.
*   **Reverse Image Search (SerpApi / Google Lens):** Safely pings out the image trace to find visual matches on the web to identify stolen assets or prior internet existence.

## Troubleshooting & Best Practices
*   **Social Media Images:** Images downloaded from modern social platforms (WhatsApp, Facebook, Instagram) are aggressively stripped of their EXIF data. Expect the OSINT Engine to return "No EXIF metadata found." You must rely on the AI and Physics engines for these files.
*   **False Positives in PRNU:** Extremely low-light images with natural high-ISO grain may sometimes trigger the PRNU irregularity warning. Always cross-compare with the ELA maps.
*   **Engine Errors:** If an engine fails, check the alert timeline; the pipeline gracefully falls back and will emit an `[ERROR]` tag identifying the specific traceback without breaking the rest of the application.
